# DuckPools Coinflip Agent Documentation

Welcome to the DuckPools Coinflip agent documentation! This directory contains comprehensive instructions and guidelines for all agent types working on the DuckPools Coinflip project.

## Available Agent Types

### Engineering Roles
- [Backend Engineer](Backend-Engineer-AGENTS.md) - Server-side development and API implementation
- [Frontend Engineer](Frontend-Engineer-AGENTS.md) - User interface and client-side development  
- [Protocol Engineer](Protocol-Engineer-AGENTS.md) - Smart contract and blockchain protocol development
- [DevOps Engineer](DevOps-Engineer-AGENTS.md) - Infrastructure, deployment, and operations
- [Ergo Specialist](Ergo-Specialist-AGENTS.md) - Ergo blockchain expertise and smart contract security

### Testing & Quality Assurance
- [QA Tester](QA-Tester-AGENTS.md) - Quality assurance and testing
- [Penetration Tester](Penetration-Tester-AGENTS.md) - Security testing and vulnerability assessment

All agent documentation files have been created and are available for reference.

## Documentation Structure

Each agent documentation follows this structure:
1. **Project Overview** - High-level description of the agent's role
2. **Domain and Responsibilities** - Key areas of focus and tasks
3. **Key Files and Directories** - Important code locations
4. **Tools to Use** - Development and testing tools
5. **Workflow** - Standard operating procedures
6. **Coding Standards** - Best practices and guidelines
7. **How to Mark Issues Done** - Completion procedures

## Getting Started

1. Choose the appropriate agent documentation based on your role
2. Review the domain, responsibilities, and workflow
3. Familiarize yourself with the key files and tools
4. Follow the coding standards and best practices
5. Use the workflow guidelines for task execution

## Contributing

If you're developing new agent types or improving existing documentation:
- Create a new `AgentType-AGENTS.md` file following the established format
- Ensure comprehensive coverage of responsibilities, tools, and workflows
- Maintain consistency with existing documentation style
- Test the agent workflows before documenting them

## Support

For questions or issues with agent documentation:
- Check the specific agent documentation first
- Contact the EM (Engineering Manager) for role-specific guidance
- Use the Paperclip issue system for documentation improvements

## Important Notes

- Documentation is living and should be updated as workflows evolve
- New agent types require comprehensive documentation before deployment
