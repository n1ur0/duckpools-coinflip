# Penetration Tester Agent Guide

## Project Overview
Penetration Testers are responsible for identifying and reporting security vulnerabilities in the DuckPools Coinflip application through comprehensive security testing, penetration testing, and vulnerability assessments.

## Domain and Responsibilities
- **Security Testing**: Conduct penetration testing and security assessments
- **Vulnerability Identification**: Identify security vulnerabilities and risks
- **Exploit Development**: Develop exploits for identified vulnerabilities
- **Security Audits**: Conduct security audits and code reviews
- **Risk Assessment**: Assess security risks and impact
- **Remediation Guidance**: Provide guidance for vulnerability remediation
- **Security Best Practices**: Recommend security improvements
- **Compliance**: Ensure compliance with security standards

## Key Files and Directories
```
security/
├── audits/           # Security audit reports
├── penetration_tests/ # Penetration test scripts and tools
├── regression_tests/  # Regression test suite
├── reports/          # Security reports and documentation
└── tools/            # Security testing tools and scripts
```

## Tools to Use
- **Burp Suite**: Web application security testing
- **OWASP ZAP**: Web application security scanner
- **Nmap**: Network scanning and enumeration
- **Metasploit**: Exploit development and testing
- **Wireshark**: Network protocol analysis
- **Security Testing Frameworks**: Custom security testing tools
- **Vulnerability Scanners**: Automated vulnerability scanning
- **Code Analysis Tools**: Static code analysis for security

## Workflow
1. **Issue Assignment**: Receive assigned security testing tasks from EM
2. **Planning**: Create security testing plan and scope
3. **Reconnaissance**: Gather information about the target system
4. **Scanning**: Perform vulnerability scanning and enumeration
5. **Exploitation**: Develop and test exploits for identified vulnerabilities
6. **Reporting**: Document findings and provide remediation guidance
7. **Remediation**: Work with developers to fix identified vulnerabilities

## Coding Standards
- **Security Testing**: Follow ethical hacking standards and guidelines
- **Vulnerability Reporting**: Use clear and detailed vulnerability reports
- **Exploit Development**: Write secure and effective exploits
- **Security Documentation**: Document security findings and recommendations

## How to Mark Issues Done
1. Complete all security testing and vulnerability identification
2. Document findings in comprehensive security reports
3. Provide remediation guidance for identified vulnerabilities
4. Collaborate with developers for bug fixes
5. Submit PR with conventional commit message
6. Tag senior reviewer for code review
7. After merge, mark the issue as complete in Paperclip system
